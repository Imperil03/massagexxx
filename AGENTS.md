# AGENTS.md

## Назначение
Эта система агентов предназначена для разработки **минимального installable PWA-хаба**: мобильной HTML-страницы с логотипом, названием проекта и набором ссылок, которую пользователь может добавить на главный экран телефона и открывать как мини-приложение.

## Главная цель
Сделать **готовый к деплою статический проект** без фреймворков и без сборки, который:
- выглядит как аккуратная мобильная страница по референсу;
- содержит набор крупных кнопок со ссылками;
- поддерживает установку на главный экран;
- открывается в standalone-режиме;
- базово работает из кэша после первого посещения.

## Архитектура агентов
Основной агент:
- `main-agent` — оркестрация, декомпозиция, сборка итогового результата.

Подагенты:
- `ui-agent` — HTML/CSS, mobile-first интерфейс, визуальная структура по референсу.
- `pwa-agent` — manifest, service worker, installability, offline shell, iOS/Android совместимость.
- `qa-agent` — финальная самопроверка, checklist, production review.

## Последовательность работы
1. `main-agent` анализирует задачу и формирует план реализации.
2. `ui-agent` делает HTML/CSS и визуальный интерфейс.
3. `pwa-agent` добавляет PWA-функциональность.
4. `main-agent` собирает итоговую структуру файлов.
5. `qa-agent` проверяет готовый результат.
6. `main-agent` выдает финальный набор файлов и краткую инструкцию.

## Обязательные файлы проекта
Ожидаемый результат должен включать:
- `index.html`
- `styles.css`
- `app.js` (если нужен)
- `manifest.webmanifest`
- `sw.js`
- каталог `icons/` с перечислением обязательных иконок

## Общие ограничения
### Не использовать
- React
- Vue
- Angular
- Svelte
- Tailwind CDN как обязательную зависимость
- npm
- сборщики
- сторонние UI-библиотеки
- Workbox
- сложную SPA-архитектуру

### Использовать
- HTML
- CSS
- Vanilla JS
- простую файловую структуру
- mobile-first верстку

## Общие требования к решению
- Код должен быть понятным и поддерживаемым.
- Не превращать страницу в маркетинговый лендинг.
- Не добавлять лишние секции, тяжелые анимации и декоративный шум.
- Приоритет: скорость, понятность, installability, удобство касания на мобильных экранах.

## Что должно быть на странице
- круглый логотип или его заглушка;
- название проекта;
- кнопка / ссылка «На сайт»;
- кнопка / ссылка «ТГ-канал»;
- кнопка / ссылка «ТП в Телеграм»;
- кнопка / ссылка «e-mail»;
- кнопка / ссылка «Чат поддержки»;
- короткая подсказка, как добавить страницу на экран телефона.

## PWA-требования
Итоговое решение должно включать:
- корректный `manifest.webmanifest`;
- `display: standalone`;
- `start_url`;
- `scope`;
- `theme_color`;
- `background_color`;
- иконки минимум `192x192` и `512x512`;
- желательно `maskable`-иконку;
- `apple-touch-icon`;
- iOS meta tags;
- service worker с базовым кэшированием app shell;
- корректную регистрацию service worker;
- install prompt через `beforeinstallprompt`, если браузер поддерживает.
- Логотип возьми в папке "logo.svg", цвет шапки: #f53277

## UX-требования
- Вся страница должна быть удобна на ширине **320–430 px**.
- Кнопки должны быть крупными, одинаковыми по стилю и расположенными вертикально.
- Не должно быть горизонтального скролла.
- Установка на экран — желательна через кнопку, но без нее страница тоже должна работать корректно.
- Ссылки должны быть вынесены в одно понятное место в коде, чтобы их было легко заменить.

## Данные-заглушки по умолчанию
Используй временные значения:
- сайт: `https://example.com`
- тг-канал: `https://t.me/example_channel`
- тп в Telegram: `https://t.me/example_support`
- e-mail: `mailto:support@example.com`
- чат поддержки: `https://example.com/support`

Название проекта:
`Массаж xXx`

## Формат итогового результата
`main-agent` должен вернуть:
1. полный код всех файлов;
2. краткую инструкцию по размещению;
3. список обязательных иконок;
4. как заменить ссылки;
5. как протестировать PWA на Android и iPhone;
6. финальный checklist готовности.

## Критерий успеха
Решение считается успешным, если пользователь может:
- открыть страницу на телефоне;
- добавить ее на главный экран;
- увидеть иконку;
- открыть страницу как мини-приложение;
- быстро перейти по нужным ссылкам.
